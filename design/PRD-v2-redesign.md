# GroTask v2.0 界面优化 PRD

> 设计稿：`design/grotask-prototype.html`
> 日期：2026-03-07

## 背景

当前 GroTask macOS 版的任务行前置了一个彩色空心圆圈，同时承担「分类指示」和「点击切换分类」两个职责。实际使用中，分类切换频率很低，但圆圈在视觉上非常抢眼，喧宾夺主，削弱了任务标题本身的信息优先级。本次优化围绕「让任务内容成为绝对主角」展开。

---

## 改动总览

| # | 改动项 | 当前行为 | 目标行为 |
|---|--------|---------|---------|
| 1 | 任务行布局 | 圆圈 + 标题 + (hover)删除 | 标题 + 色点 + (hover)删除 |
| 2 | 分类指示 | 前置 20px 彩色空心圆圈 | 后置 7px 实心小色点，半透明 |
| 3 | 完成交互 | 点击圆圈完成（行点击无响应） | 点击整行完成任务 |
| 4 | 已完成行 | 绿色空心勾 + 删除线标题 | 绿色实心勾图标 + 删除线标题 + 完成时间 |
| 5 | Panel 高度 | 固定高度 | 动态高度，根据内容自适应（设最大高度） |

---

## 1. 任务行布局重构

### 1.1 未完成任务行

```
┌─────────────────────────────────────────────────┐
│  任务标题文字                          ● (hover)✕ │
│                                      色点  删除  │
└─────────────────────────────────────────────────┘
```

**结构（从左到右）：**
- 任务标题（`flex: 1`，单行省略，最多两行）
- 分类色点（7px 实心圆，蓝色=工作，橙色=生活，opacity 0.5）
- 置顶图标（仅置顶任务显示，灰色小图标）
- 删除按钮（hover 时才显示，红色 ✕）

**交互：**
- **点击整行** → 完成任务（带 spring 动画）
- **右键菜单** → 编辑 / 置顶 / 切换分类 / 时间视角 / 删除（保持不变）
- **hover** → 行背景高亮 + 显示删除按钮 + 色点透明度提升

**设计意图：** 去掉前置圆圈后，用户视线直接落在标题上，阅读效率提升。分类信息降级为辅助提示。

### 1.2 已完成任务行

```
┌─────────────────────────────────────────────────┐
│  ✓  任务标题（删除线）                             │
│     完成时间                                      │
└─────────────────────────────────────────────────┘
```

**结构（从左到右）：**
- 绿色实心勾图标（16px，`checkmark.circle.fill`）
- 内容区：
  - 标题（删除线，降低透明度）
  - 完成时间（小字灰色，如 "14:32" 或 "03/06 20:15"）

**交互：**
- **点击整行** → 恢复为未完成（带 spring 动画）
- 行背景带淡绿色底色

## 2. 分类指示方式变更

### 当前
- 前置 20px 空心彩色圆圈
- 点击圆圈可切换工作/生活分类
- 视觉权重高，与标题争夺注意力

### 目标
- 后置 7px 实心小色点
- 默认 opacity 0.5，hover 时 0.7
- 不可直接点击切换分类（分类切换移至右键菜单，已有此功能）

### 颜色映射（不变）
- 工作：`Color(.systemBlue)` / #007AFF
- 生活：`Color(.systemOrange)` / #FF9500

---

## 3. 完成任务交互变更

### 当前
- 点击行前的圆圈图标 → 完成任务
- 点击行本身无响应（仅右键有菜单）

### 目标
- **点击任务行任意位置** → 完成/恢复任务
- 动画：`spring(response: 0.25, dampingFraction: 0.75)`
- 完成时自动记录 `completedAt`，恢复时清空

### 注意事项
- 删除按钮需 `stopPropagation`，防止误触完成
- 编辑状态下点击行不应触发完成
- 右键菜单中的「完成」入口保留

---

## 4. Panel 动态高度

### 当前
- Panel 高度固定（通过 `frame(width: 320)` 未指定高度，但内容区无限制）

### 目标
- Panel 高度根据内容自适应
- **最小高度**：约 200px（header + tabs + 空状态 + input + footer）
- **最大高度**：480px（超过后内容区滚动）
- 内容增减时高度平滑过渡（`animation: .easeInOut(duration: 0.2)`）

### 实现建议
- 移除任务列表的固定高度
- 使用 `frame(maxHeight: 360)` 限制列表区域最大高度
- 列表区域内容不足时自然收缩
- 整个 Panel 使用 `fixedSize(horizontal: false, vertical: true)` 或类似方式

---

## 5. 不变的部分

以下保持现有行为不变：
- Header（标题 + 添加按钮）
- Scope Tab Bar（快速/今天/随时/将来/全部）及其筛选逻辑
- 快速添加输入栏（分类色点 + scope 图标 + 输入框）
- 右键菜单（编辑/置顶/切换分类/时间视角/删除）
- Section 折叠展开逻辑
- 置顶 section 在筛选视图中始终显示
- 底部「退出 GroTask」
- CloudKit 同步、数据模型

## 6. 迁移注意

### macOS 端
- `TaskRowView.swift` — 重构行布局，移除 `leadingButton`，添加 trailing 色点
- `TaskPopoverView.swift` — 行点击事件从圆圈移到整行
- `FloatingPanel.swift` — 动态高度适配

### iOS 端
- iOS 端使用 swipe actions，行布局同步调整（标题前移，色点后置）
- 点击行完成任务的交互需评估是否适合 iOS（iOS 通常用 swipe 或 leading button）
